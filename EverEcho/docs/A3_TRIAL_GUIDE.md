### 阶段 2: 试用执行（Day 1-3）

#### 2.1 试用者自主试用

**试用任务**:

1. **必做任务**（30 分钟）
   - [ ] 连接钱包
   - [ ] 注册账户
   - [ ] 发布一个任务
   - [ ] 接受一个任务（需要第二个账户）
   - [ ] 完成任务流程

2. **可选任务**（30 分钟）
   - [ ] 尝试 Request Fix
   - [ ] 尝试协商终止
   - [ ] 查看个人资料
   - [ ] 浏览任务广场

#### 2.2 实时支持

**支持时间**: 每天 10:00-22:00

**支持方式**:
- 微信群/Telegram 群实时答疑
- 问题收集表记录
- 必要时远程协助

#### 2.3 问题收集

**收集方式**:
1. 试用者填写问题收集表
2. 群内讨论记录
3. 截图和录屏

**收集内容**:
- 遇到的问题
- 不理解的地方
- 改进建议
- 整体体验评分

---

### 阶段 3: 问题整理（Day 4）

#### 3.1 问题分类

将收集到的问题分类：

1. **阻塞性问题**（P0）
   - 无法完成核心流程
   - 交易失败
   - 页面崩溃

2. **体验问题**（P1）
   - 操作不流畅
   - 提示不清晰
   - 加载时间长

3. **文档问题**（P2）
   - 文档不清楚
   - 缺少说明
   - 步骤遗漏

4. **改进建议**（P3）
   - 功能建议
   - UI 优化
   - 流程优化

#### 3.2 问题归属

确定每个问题的归属层：

- **合约层**: 合约逻辑、Gas 费用
- **后端层**: API 错误、数据库问题
- **前端层**: UI 问题、交互问题
- **文档层**: 文档不清晰、步骤遗漏

---

### 阶段 4: 快速修复（Day 5）

#### 4.1 修复阻塞性问题

**原则**: 只做薄片级别补丁，不引入新需求

**允许的修复**:
- 修复 Bug
- 优化提示文案
- 补充文档说明
- 添加错误处理

**不允许的修复**:
- 改变业务逻辑
- 添加新功能
- 重构代码
- UI 美化（除非必需）

#### 4.2 记录 Patch

所有修复记录到 `A3_PATCH_NOTES.md`:

```markdown
## Patch A3-001: 修复注册失败提示不清晰

**问题**: 注册失败时只显示 "Transaction failed"
**归属**: 前端层
**优先级**: P1
**修复**: 添加详细错误提示，区分不同失败原因
**文件**: frontend/src/hooks/useRegister.ts
**验证**: 测试各种失败场景
```
