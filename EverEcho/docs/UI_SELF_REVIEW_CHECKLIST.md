## 使用说明

本清单用于在真人试用前快速检查 UI 合理性。每个页面 6-10 条自检项，逐条确认。

**检查标准**:
- ✅ 通过 - 符合预期
- ⚠️ 需改进 - 可用但体验不佳
- ❌ 不通过 - 阻塞试用

---

## 1. Home 页面

### 信息层级
- [ ] 页面标题清晰（EverEcho / 欢迎信息）
- [ ] 功能介绍简洁易懂（3-5 句话）
- [ ] 主要 CTA 明显（Connect Wallet 按钮）

### 状态处理
- [ ] 未连接钱包：显示 Connect Wallet 按钮
- [ ] 已连接未注册：自动跳转到 Register 或显示引导
- [ ] 已连接已注册：显示进入任务广场的入口

### 用户引导
- [ ] 新用户能理解下一步要做什么
- [ ] 网络提示清晰（Sepolia 测试网）
- [ ] 错误提示可见（连接失败、网络错误）

## 2. Register 页面

### 表单可用性
- [ ] 所有必填字段有 * 标记
- [ ] 输入框 placeholder 清晰
- [ ] 技能选择至少 2 个的提示明显
- [ ] 表单验证实时反馈（红色错误提示）

### 交易流程
- [ ] 提交按钮禁用状态清晰（loading 时）
- [ ] MetaMask 弹窗提示友好
- [ ] 交易发送后显示 tx hash
- [ ] 交易确认后显示成功消息

### 特殊情况
- [ ] CAP 满时提示清晰（"CAP reached, no EOCHO minted"）
- [ ] 交易失败时错误可见且可复制
- [ ] 成功后自动跳转到任务广场

## 8. 移动端适配

### 响应式布局
- [ ] 主要页面在 375px 宽度可用

- [ ] 列表可滚动

### 触摸友好
- [ ] 按钮尺寸 ≥ 44px

## 9. 错误处理

### 前端错误
- [ ] 所有错误通过 Toast 或 Alert 显示
- [ ] 错误消息用户友好（非技术术语）
- [ ] 错误详情可展开和复制
- [ ] 提供下一步操作建议

### 后端错误
- [ ] API 错误有明确提示
- [ ] 网络错误有重试入口
- [ ] 元数据加载失败不阻塞主流程

### 合约错误
- [ ] 用户拒绝交易：友好提示
- [ ] 余额不足：明确说明
- [ ] Gas 估算失败：提供建议
- [ ] 合约执行失败：显示原因

---

## 10. 性能与稳定性

### 加载性能
- [ ] 页面首次加载 < 3 秒
- [ ] 列表加载有 Loading 提示
- [ ] 无明显卡顿

### 防抖与节流
- [ ] 按钮重复点击有防抖
- [ ] 表单提交有 loading 状态
- [ ] 轮询间隔合理（3-5 秒）

### 错误恢复
- [ ] RPC 错误可重试
- [ ] 元数据加载失败可重试
- [ ] 页面刷新后状态保持
