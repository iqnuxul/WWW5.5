# 🌸 SheAid：女性安全捐助链白皮书

版本号：v1.0（2025 Q4）

作者团队：SheAid Foundation（研发中）

关键词：Web3、女性援助、隐私保护、反腐败、NGO、公益链、价格Oracle、去信任救助

---

一、前言
全球范围内，慈善项目常面临三大难题：
1. 账目不公开、运营不透明，导致社会信任危机；
2. 采购环节存在价格虚高与利益输送，善款浪费严重；
3. 受助人（尤其是女性与未成年人）在接受帮助时隐私易被泄露，甚至遭到家暴或剥夺。
SheAid 致力于改变这一现状。
 我们通过将 Web3 的可验证性（Verifiability） 与 社会组织的温度（Humanity） 相结合，构建一条能让善款透明流转、隐私受保护、援助真正到达弱势女性手中的 去信任捐助链。

---

二、愿景与使命

愿景：
让每一笔善款都安全、透明地到达真正需要帮助的女性手中。
使命：
1. 以区块链技术重塑慈善透明度，消除“账目黑箱”；
2. 用隐私保护机制防止受助女性因性别与家庭环境遭受二次伤害；
3. 引入智能合约自动审计与挑战奖励机制，减少人为腐败与操作空间；
4. 以女性社群为核心，构建去中心化的善意网络。

三、现存问题分析
|问题类别	|传统体系痛点	|社会后果|
|----|----|----|
|资金透明度	|财务人工录入、数据滞后、易被篡改	|捐助人信任度下降|
|采购腐败	|缺乏价格比较、招标不透明	|善款浪费、价格虚高|
|物资挪用	|家庭内被男性成员夺取、未成年人被剥夺	|援助目标失效|
|隐私安全	|名单公开、身份暴露	|受助人遭报复或社会歧视|
|审计缺失	|人治、无激励举报机制	|容易掩盖问题|


---
四、SheAid 设计原则
1. 最小信任原则（Minimal Trust）：每笔支出在链上验证合规，任何人可追溯。
2. 隐私优先原则（Privacy First）：受助人身份仅由 NGO 授权方掌握，链上使用匿名哈希。
3. 反腐自动化原则（Auto-Audit）：智能合约强制价格带、时间锁、公示与挑战。
4. 温度与制度并存（Tech + Humanity）：女性志愿者、NGO 保障线下温度与安全。
5. 低Gas与可持续性：仅核心逻辑上链，其余数据以加密哈希存证。
